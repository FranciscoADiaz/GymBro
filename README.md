
QUÉ ES?
Es un starter full-stack MERN con:
Backend Node/Express + MongoDB, JWT middleware base, CORS/helmet/morgan y estructura en capas (controllers/services/models/routes).
Frontend Vite/React con Router, AuthContext, Axios preconfig, rutas privadas y tests de arranque.
Lint/Prettier y tests (Jest/Supertest en back, Vitest/RTL en front) para mantener calidad.
Te sirve para proyectos de tipo SPA + API REST: paneles/admin, dashboards, apps con login/token, CRUD sobre Mongo, etc. Solo agrega los casos de negocio (users, pagos, etc.) y endpoints reales.
# Boilerplate-MERN


CÓMO USARLO DESPUÉS?
En lugar de clonar o copiar carpetas, conviertes tu repo del Boilerplate en un "Molde".

Cómo se hace (Es muy fácil):
Subes tu código (Front y Back) como un Repositorio normal, en GitHub, vas a la pestaña Settings (Configuración) de ese repositorio.
Justo al principio, verás una casilla que dice: "Template repository", márcala.

¿Qué ganas con esto?
La próxima vez que tengas un cliente nuevo, no tendrás que clonar ni borrar carpetas .git.
En vez del botón de "Clone", verás un botón verde enorme que dice: "Use this template" (Usar esta plantilla).
Le das clic, le pones nombre nuevo (ej: Gym-Cliente-X) y GitHub te crea un repo nuevo, con todo tu código copiado, pero con el historial limpio y listo para empezar.
